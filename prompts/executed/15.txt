Implement Leira v1.4: Filesystem Workspace

Mission

Extend Leira from a machine that records artifacts into a machine that stores evidence.

The goal is not synchronization.

The goal is not cloud storage.

The goal is to prove that artifacts may exist outside the ledger while remaining mechanically verifiable.

The machine must be able to say:

> A worker produced bytes.
>
> Those bytes were written to a file.
>
> Here is where they live.
>
> Here is their hash.
>
> Here is their size.
>
> Here is the receipt that references them.

Nothing more.

* Python only.

* Library, not service.
* No threads.
* No async.

Adapt existing structure minimally.

Artifact Descriptor

Implement:

```python
@dataclass(frozen=True)
class ArtifactDescriptor:
    artifact_id: str
    intent_id: str
    relative_path: str
    sha256: str
    size_bytes: int
    created_at: str
```

Rules

Artifacts are immutable.

Artifact bytes are external.

Metadata lives in the ledger.

The filesystem is evidence storage.

The ledger remains truth.

Hashing

Use:

```python
sha256(content)
```

Store:

```python
sha256
size_bytes
```

Rules

Hash is computed from bytes.

Never trust filesystem timestamps.

Never trust filenames.

Hash is evidence.

Filesystem Safety

Treat:

```python
relative_path
```

as untrusted input.

Reject:

```text
..
absolute paths
empty paths
```

Normalize paths.

Require paths to remain inside:

```text
workspace/intents/<intent_id>/artifacts
```

Suggested errors:

```python
INVALID_PATH
PATH_TRAVERSAL
```

Allowed characters may be restricted to boring filenames.

The workspace is an evidence locker.

Not a shell.

Filesystem Writes

Use exclusive binary mode:

```python
open(path, "xb")
```

Rules

No overwrite.

No append mode.

No mutation.

No delete API.

No rename API.

Artifacts are write-once.

If file already exists:

append:

```text
artifact_write_rejected
```

Return:

```python
ALREADY_EXISTS
```

Artifact Event

Successful write appends:

```text
artifact_file_written
```

Artifact schema:

```python
{
    "type": "artifact_file",
    "content": {
        "artifact_id": "...",
        "intent_id": "...",
        "relative_path": "...",
        "sha256": "...",
        "size_bytes": 1234
    }
}
```

Rejected writes append:

```text
artifact_write_rejected
```

with:

```python
error_type
```

Receipt Integration

Receipt bundles should contain:

```python
ArtifactDescriptor
```

references.

Do not embed file bytes.

Receipts point to evidence.

They do not contain evidence.

Projection

Create:

```text
artifact_projection
```

Fields:

```python
artifact_id
intent_id
relative_path
sha256
size_bytes
created_at
last_event_id
```

Projection is derived.

created_at comes from ledger timestamps.

Never use:

```python
datetime.now()
```

during rebuild.

Projection Rebuild

Extend rebuild logic.

Scan:

```text
artifact_file_written
```

events.

Reconstruct projection.

Rebuild must be:

* deterministic
* idempotent
* transactional

Audit Extension

Audit now performs filesystem I/O.

This is acceptable.

Truth outranks speed.

Verify:

* file exists
* file size matches size_bytes
* file hash matches sha256
* artifact projection matches ledger

Distinguish failures:

Missing file:

```text
MISSING_ARTIFACT_FILE
```

File exists but size differs:

```text
SIZE_MISMATCH
```

File exists and size matches but hash differs:

```text
HASH_MISMATCH
```

Audit remains read-only.

Audit reports.

It does not repair.

Failure Model

Missing files are recoverable by humans.

File corruption is detectable.

Ledger corruption is not.

Do not optimize audit.

Slow verification is honest verification.

Kernel Rules

The workspace does not:

* synchronize
* upload
* delete
* compress
* version
* rename
* mutate files

It writes files once.

It records evidence.
