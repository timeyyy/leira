Implement Leira v1.5: Git Provenance

Mission

Extend Leira from a machine that stores evidence into a machine that records the source state behind evidence.

The goal is not version control.

The goal is not source management.

The goal is not synchronization.

The goal is to prove that artifacts can be tied to the repository state that existed before execution began.

The machine must be able to say:

> These artifacts were produced.
>
> This source state existed before execution.
>
> This commit was active.
>
> The repository was clean or dirty.
>
> Provenance was captured or provenance capture failed.
>
> Here is the evidence linking intent, execution, artifacts, and source.

Nothing more.

Provenance Snapshot

Implement:

```python
@dataclass(frozen=True)
class ProvenanceSnapshot:
    snapshot_id: str
    intent_id: str
    repo_path: str
    head_sha: str | None
    branch: str | None
    is_dirty: bool | None
    status_porcelain: str
    created_at: str
    error_type: str | None = None
    stderr: str = ""
```

Rules

Snapshots are immutable.

Git is observed.

Not modified.

Git state is evidence.

Not truth itself.

Ledger remains truth.

Store:

```python
status_porcelain
```

exactly as Git returned it.

Do not parse.

Do not filter.

Do not reinterpret.

Snapshot API

Implement:

```python
capture_provenance(
    intent_id: str,
    repo_path: str
) -> ProvenanceSnapshot
```

Implement:

```python
get_provenance(
    snapshot_id: str
) -> ProvenanceSnapshot | None
```

Reuse:

```python
inspect_repo()
```

from the existing Git adapter.

Do not duplicate subprocess logic.

Snapshot Timing

Capture provenance immediately before worker execution.

This represents:

```text
source state at execution start
```

Do not capture after execution.

Do not update snapshots after execution.

Worker side-effects are irrelevant to the captured source state.

Historical truth outranks current state.

Successful Capture Event

Successful capture appends:

```text
provenance_captured
```

Artifact schema:

```python
{
    "type": "provenance",
    "content": {
        "snapshot_id": "...",
        "intent_id": "...",
        "repo_path": "...",
        "head_sha": "...",
        "branch": "...",
        "is_dirty": false,
        "status_porcelain": "",
        "error_type": None,
        "stderr": ""
    }
}
```

Failed Capture Event

If Git inspection fails:

append:

```text
provenance_capture_failed
```

Artifact schema:

```python
{
    "type": "provenance",
    "content": {
        "snapshot_id": "...",
        "intent_id": "...",
        "repo_path": "...",
        "head_sha": None,
        "branch": None,
        "is_dirty": None,
        "status_porcelain": "",
        "error_type": "NOT_REPOSITORY",
        "stderr": "..."
    }
}
```

Rules for failure:

* failed capture is history
* failed capture is not hidden
* failed capture prevents worker execution
* do not proceed to dispatch if provenance could not be captured
* return typed failure

Suggested errors:

```text
NOT_REPOSITORY
GIT_NOT_FOUND
TIMEOUT
UNEXPECTED
STORAGE_FAILURE
```

Dispatcher Integration

Add provenance capture as a pre-execution step.

Sequence:

1. intent is claimed
2. provenance is captured
3. if provenance capture fails:
   * append provenance_capture_failed
   * mark intent_failed
   * release claim if possible
   * do not invoke worker
4. if provenance capture succeeds:
   * append provenance_captured
   * invoke worker
   * continue normal dispatch flow

Provenance failure is not worker failure.

It is execution precondition failure.

Kernel Rules

The kernel does not:

* interpret branches
* judge dirty repositories
* require clean repositories
* reject dirty repositories
* compare commits
* inspect diffs
* audit against current repo state

It records:

* commit
* branch
* dirty flag
* porcelain output
* capture error if any

Nothing more.

Dirty Repository Rule

Dirty repositories are allowed.

Clean repositories are allowed.

The system records dirtiness.

It does not judge it.

Projection is derived.

created_at comes from ledger timestamps.

Never use:

```python
datetime.now()
```

during rebuild.

Projection Rebuild

Scan:

```text
provenance_captured
provenance_capture_failed
```

events.

Rebuild must be:

* deterministic
* idempotent
* transactional

Successful and failed provenance events both appear in projection.

Audit Extension

Verify:

* provenance projection matches ledger
* captured head_sha matches captured event
* branch matches captured event
* dirty flag matches captured event
* status_porcelain matches captured event exactly
* failed captures preserve error_type
* failed captures preserve stderr
* snapshot ids are unique
* receipts reference valid snapshots
* provenance created_at equals ledger event timestamp

Audit remains read-only.

Audit does not compare against the current repository.

The repository may legitimately evolve.

Truth is historical.

Not present state.

Failure Model

Repository missing after execution is acceptable.

Commit deleted later is acceptable.

Branch renamed later is acceptable.

Repository dirty state changing later is acceptable.

Provenance captures observed history.

Not current reality.

Current repository state does not rewrite history.

Nothing is hidden.
