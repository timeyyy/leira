Implement Leira v1.6: Session Bundles

Mission

Extend Leira from a machine that records individual intents into a machine that can correlate related work.

The goal is not orchestration.

The goal is not planning.

The goal is not project management.

The goal is to prove that multiple intents, receipts, artifacts, and provenance snapshots can be grouped into a higher-level evidence set without changing the underlying truth model.

The machine must be able to say:

> These intents belonged to the same session.
>
> These receipts belong together.
>
> These artifacts belong together.
>
> These provenance snapshots belong together.
>
> Here is the evidence.
>
> No execution order was enforced.

Nothing more.

Session Bundle

Implement:

```python
@dataclass(frozen=True)
class SessionBundle:
    session_id: str
    created_at: str
    intent_ids: list[str]
```

Rules

Sessions are views.

Sessions are additive.

Sessions do not own truth.

Intents remain independent.

The ledger remains authoritative.

Projection

Create:

```text
session_projection
```

Fields:

```python
session_id
intent_count
created_at
last_event_id
```

Create:

```text
session_membership_projection
```

Fields:

```python
session_id
intent_id
membership_order
created_at
last_event_id
```

Rules

created_at derives from ledger timestamps.

membership_order derives from ledger order.

membership_order is an incrementing integer per session_id.

Never use:

```python
datetime.now()
```

during rebuild.

Projection Rebuild

Scan ledger events chronologically:

```text
session_created
session_intent_added
```

For each:

```text
session_intent_added
```

calculate:

```python
membership_order = current_count_for_session + 1
```

Rebuild must be:

* deterministic
* idempotent
* transactional

Deleting projections loses no truth.

Audit Extension

Verify:

* session_projection matches ledger
* session_membership_projection matches ledger
* membership_order is preserved exactly
* membership_order is derived from ledger order
* duplicate memberships detected
* unknown session references detected
* unknown intent references detected
* receipt membership queries match session members
* artifact membership queries match session members
* provenance membership queries match session members

Audit remains read-only.

Audit reports disagreement.

It does not repair.

Failure Model

Sessions are convenience.

Session projections are a table of contents.

Deleting projections loses query convenience.

It does not lose work.

Session corruption is recoverable.

Ledger corruption is not.
