Implement Leira v1.7: Archive & Replay

Mission

Extend Leira from a machine that correlates evidence into a machine that can move without losing truth.

The goal is not backup.

The goal is not synchronization.

The goal is not restoration from opaque state.

The goal is to prove that an entire workshop history can be exported, transported, imported, and replayed into an empty machine.

The machine must be able to say:

> Here is the history.
>
> Here are the files.
>
> Here are the receipts.
>
> Here are the projections rebuilt from history.
>
> This machine was reconstructed from evidence.
>
> No worker was executed during replay.

Nothing more.

Archive Bundle

Implement:

```python
@dataclass(frozen=True)
class ArchiveBundle:
    created_at: str
    first_event_id: str
    last_event_id: str
    event_count: int
    archive_sha256: str
```

Archive API

Implement:

```python
export_archive(
    output_path: Path
) -> ArchiveBundle
```

Implement:

```python
import_archive(
    archive_path: Path
) -> ArchiveBundle
```

Implement:

```python
replay_history()
```

Archive Format

Create:

```text
archive/
  manifest.json
  ledger_events.jsonl
  workspace/
    intents/
      <intent_id>/
        artifacts/
          <files>
```

Manifest contains:

```python
{
    "created_at": "...",
    "first_event_id": "...",
    "last_event_id": "...",
    "event_count": 123,
    "ledger_events_sha256": "...",
    "workspace_sha256": "..."
}
```

Ledger JSONL

Use:

```text
ledger_events.jsonl
```

Rules:

* one event per line
* ledger order preserved
* no timestamp sorting
* no pretty printing
* deterministic JSON per line

Use canonical JSON:

```python
json.dumps(
    event,
    sort_keys=True,
    separators=(",", ":")
)
```

History outranks clocks.

Artifact Rules

Archives include:

* ArtifactDescriptor metadata from ledger/projections
* file bytes from workspace

On export:

* copy workspace files into archive workspace structure exactly
* preserve relative paths
* verify file sha256 before export
* fail export if file hash does not match ledger

On import:

* recreate workspace folder structure exactly
* write files using exclusive write mode where possible
* verify imported file sha256 after copy
* never mutate existing files silently

Replay Rules

Replay starts from:

```text
empty SQLite database
empty projection tables
empty or target workspace
```

Replay sequence:

1. initialize schema
2. import ledger_events.jsonl into ledger_events in ledger order
3. validate_chain()
4. restore workspace files
5. rebuild all projections
6. run audit()

Replay must be:

* deterministic
* idempotent where safe
* transactional for database import
* non-executing

Replay Immutability

During replay, disable:

* dispatcher
* worker invocation
* shell adapter execution
* Git inspection
* provenance recapture

Replay must never:

* execute workers
* rerun shell commands
* query current Git repositories
* regenerate artifacts
* reinterpret events
* repair history

Replay reconstructs views.

Nothing more.

Kernel Rules

Archive does not:

* compress
* encrypt
* synchronize
* upload
* repair
* summarize
* explain

Replay does not:

* execute
* think
* inspect current source
* regenerate output

Archive is a serializable record of reality.

Replay re-derives representation from history.

Projection Rebuild

After ledger import, rebuild:

* operation_state_projection
* intent_projection
* worker_projection
* intent_claim_projection
* receipt_projection
* artifact_projection
* provenance_projection
* session_projection
* session_membership_projection

All rebuilds must use ledger timestamps.

Never use:

```python
datetime.now()
```

during replay-derived rebuilds.

Audit Extension

Verify:

* replayed database matches archive manifest
* event_count matches
* first_event_id matches
* last_event_id matches
* ledger_events_sha256 matches
* receipt counts match
* session memberships match
* artifact descriptors match
* artifact hashes match
* provenance snapshots match
* workspace files exist
* validate_chain() succeeds

Audit remains read-only.

Export Determinism

Exporting the same machine state twice should produce byte-identical:

```text
ledger_events.jsonl
```

and deterministic manifest fields except:

```text
created_at
archive_sha256
```

If testing full archive hash determinism, allow passing fixed created_at.
