Implement Leira v2.1: OpenAI Mind

Mission

Extend Leira from a machine that hosts minds into a machine that can host its first external API-backed intelligence.

The goal is not agents.

The goal is not orchestration.

The goal is not memory.

The goal is not prompt engineering.

The goal is to prove that OpenAI can participate as a Mind while preserving the same mechanical guarantees as EchoMind.

The machine must be able to say:

> A request was made.
>
> OpenAI responded or failed.
>
> Here is exactly what was requested.
>
> Here is exactly what was returned.
>
> Here is which model was configured.
>
> No secret was recorded.
>
> The kernel does not know that OpenAI exists.

Nothing more.

Implementation

Implement:

```python
class OpenAIMind(Mind):
```

Constructor:

```python
OpenAIMind(
    model: str,
    api_key: str,
    timeout_seconds: int = 60
)
```

Properties:

```python
name: str
```

Suggested name:

```python
OpenAI:<model>
```

Invoke:

```python
invoke(
    request: MindRequest
) -> MindResponse
```

Request

Input:

```python
MindRequest(
    text="..."
)
```

Output:

```python
MindResponse(
    success=True,
    text="..."
)
```

Rules

One request.

One response.

No streaming.

No conversation history.

No hidden context.

No retries.

No interpretation.

No prompt templates.

No automatic system prompt.

If the API requires a message structure, wrap only the provided request text.

Adapter Boundary

The OpenAI adapter owns:

* client initialization
* HTTP request
* timeout handling
* API failure mapping
* response parsing
* partial/empty response handling

The kernel owns:

* recording
* receipts
* projections
* audit mechanics

The kernel must not import OpenAI.

The dispatcher must not import OpenAI.

The worker protocol must not import OpenAI.

Failure Handling

All operational failures return:

```python
MindResponse(
    success=False,
    text="",
    error_type="...",
    error_message="..."
)
```

Never raise operational exceptions from the adapter.

Suggested error types:

```text
AUTH_FAILURE
RATE_LIMIT
TIMEOUT
NETWORK_FAILURE
POLICY_VIOLATION
EMPTY_RESPONSE
UNEXPECTED_RESPONSE
UNEXPECTED
```

Map moderation, safety, or policy refusals/errors to:

```text
POLICY_VIOLATION
```

Map malformed API responses to:

```text
UNEXPECTED_RESPONSE
```

Map empty model output to:

```text
EMPTY_RESPONSE
```

Secret Hygiene

Never record:

* api_key
* Authorization headers
* raw client config containing secrets
* environment variables containing secrets

Allowed to record:

* mind_name
* model
* request text
* response text
* error_type
* error_message if scrubbed of secrets

Before storing error_message, scrub obvious secret-bearing substrings.

Model Provenance

Record the configured model name as provenance metadata in the mind output.

Example worker output:

```python
{
    "mind_name": "OpenAI:gpt-...",
    "model": "gpt-...",
    "request": "...",
    "response": "..."
}
```

Environment Integration

Environment snapshots may capture:

```text
OPENAI_MODEL
```

or configured model name.

They must never capture:

```text
OPENAI_API_KEY
```

or any secret-bearing environment variable.

Provenance Boundary

Git provenance records local source code.

Environment provenance records local runtime context.

OpenAI model provenance records configured model name only.

Leira does not attempt to prove:

* remote model weights
* OpenAI internal state
* training data
* server-side implementation

Artifact Path

Responses appear through ordinary:

```text

```

artifacts via MindWorker.

No special OpenAI artifact type is required.

No special OpenAI lifecycle exists.

```python
{
    "type": "worker_result",
    "content": {
        "intent_id": "...",
        "worker_name": "OpenAI:gpt-...",

        "outputs": {

            "request": "hello",
            "response": "hello"
        },
        "worker_success": True,
        "error_type": None,
        "error_message": ""
    }
}
```
