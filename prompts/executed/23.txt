Implement Leira v2.3: Claude Mind

Mission

Extend Leira from a machine that hosts one external intelligence into a machine that hosts multiple external intelligences without changing the kernel.

The goal is not multi-agent systems.

The goal is not consensus.

The goal is not orchestration.

The goal is to prove that ClaudeMind is merely another guest conforming to the existing Mind protocol.

The machine must be able to say:

> OpenAIMind was registered.
>
> ClaudeMind was registered.
>
> EchoMind was registered.
>
> The caller chose ClaudeMind.
>
> The registry resolved ClaudeMind.
>
> MindWorker wrapped ClaudeMind.
>
> The dispatcher executed a Worker.
>
> The kernel did not know which mind was used.

Nothing more.

Failure Handling

All operational failures return:

```python
MindResponse(
    success=False,
    text="",
    error_type="...",
    error_message="..."
)
```

Never raise operational exceptions from the adapter.

Suggested error types:

```text
AUTH_FAILURE
RATE_LIMIT
TIMEOUT
NETWORK_FAILURE
POLICY_VIOLATION
EMPTY_RESPONSE
UNEXPECTED_RESPONSE
UNEXPECTED
```

Map Anthropic/provider-specific exceptions inside the adapter.

Examples:

```text
anthropic.AuthenticationError -> AUTH_FAILURE
anthropic.RateLimitError -> RATE_LIMIT
anthropic.APITimeoutError -> TIMEOUT
anthropic.APIConnectionError -> NETWORK_FAILURE
safety/moderation/refusal/policy errors -> POLICY_VIOLATION
malformed API responses -> UNEXPECTED_RESPONSE
empty model output -> EMPTY_RESPONSE
unknown exceptions -> UNEXPECTED
```

Provider-specific errors must never leak into kernel logic.

Secret Hygiene

Never record:

* api_key
* Authorization headers
* x-api-key headers
* raw client config containing secrets
* environment variables containing secrets

Allowed to record:

* mind_name
* model
* request text
* response text
* error_type
* scrubbed error_message

Before returning error_message, scrub obvious secret-bearing substrings.

Do not expose the API key through:

```python
__repr__
__str__
dataclass fields
public attributes
serialized state
```

Store secrets in private attributes only, and do not include them in outputs.
