# Leira v1.15 — Human Decision Record

## Mission

Implement the deterministic **Human Decision Record** slice.

This slice preserves an explicit human decision about the current lifecycle state.

It does **not** execute the decision.

It does **not** dispatch work.

It does **not** modify project state.

It merely records that a human made a decision, why they made it, and which evidence informed it.

The lifecycle becomes:

```text
Project State
↓

Prompt Draft
↓

Knowledge Gap
↓

Review Question
↓

Review Record
↓

Prompt Revision Record
↓

Human Decision Record
```

---

# Architectural Review

Leira now preserves:

* uncertainty
* questions
* review evidence
* prompt evolution

The next missing piece is the human judgement that determines what should happen next.

This decision is itself evidence.

The dispatcher will eventually execute decisions.

It must never invent them.

# Public API

Create immutable dataclasses.

## HumanDecision

Fields:

```text
decision_id

subject_id

subject_kind

decision_type

reason_codes

source_label

review_record_ids

prompt_revision_id

decision_summary
```

All fields are caller supplied.

No generated identifiers.

No timestamps.

No clocks.

No UUIDs.

No randomness.

---

## Supported decision types

The module stores the supplied decision only.

Suggested supported values:

```text
approve

request_review

continue_planning

implement

defer

reject

cancel

custom
```

Unknown values are preserved exactly.

The module performs no validation beyond deterministic storage.

# Steward's Law

Keep it boring.

The kernel should preserve the fact that a human made a decision.

The kernel must remain completely ignorant of what happens because of that decision.

Execution belongs to future deterministic dispatch layers.
