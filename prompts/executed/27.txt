Implement Leira v2.7: Status View

Mission

Extend Leira from a machine that explains history into a machine that can expose current state.

The goal is not dashboards.

The goal is not monitoring.

The goal is not notifications.

The goal is to provide a deterministic, read-only view of the workshop.

The machine must be able to say:

> These are the current intents.
>
> These are the current claims.
>
> These are the current sessions.
>
> These are the current minds.
>
> These are the current proposals.
>
> No state was changed.
>
> No recommendation was made.

Nothing more.

Rules

StatusView is derived.

StatusView is disposable.

StatusView is not stored.

StatusView is not authoritative.

The ledger remains truth.

The projections serve the view.

Nothing more.

Data Sources

StatusView must read only from projections:

* intent_projection
* intent_claim_projection
* session_projection
* proposal_projection
* proposal_decision_projection
* mind_projection

Never scan current filesystem.

Never inspect current Git state.

Never invoke workers.

Never rebuild projections.

Never walk the ledger as the primary source for current state.

Never inspect live environment.

Snapshot Read Rule

Generate the view inside a single read transaction where practical.

The view should observe a consistent database snapshot.

Do not hold write locks.

Do not block execution longer than necessary.

Generated Timestamp

The only new timestamp allowed is:

```python
generated_at
```

Allow fixed timestamp injection for deterministic tests.

If omitted, generate current UTC timestamp.

Projection Semantics

StatusView reflects the known projected state.

If the ledger contains data that has not been rebuilt or projected, StatusView should not infer it.

That is correct.

Views observe projections.

Auditors verify projections.

Rebuilders repair projections.

StatusView does none of those jobs.

Proposal Status Rule

Proposal status should be derived mechanically:

* if proposal has no decision, status is PENDING
* if proposal_decision_projection says APPROVED, status is APPROVED
* if proposal_decision_projection says REJECTED, status is REJECTED

Do not infer from execution.

Do not infer from inbox.

Do not infer from claims.
