# Leira v1.16 — Dispatch Record

## Mission

Implement the deterministic **Dispatch Record** slice.

This slice records an intended dispatch action derived from a Human Decision.

It does **not** execute the action.

It does **not** call tools.

It does **not** run prompts.

It does **not** modify project state.

It merely preserves:

* which human decision authorized the dispatch,
* what artifact is being dispatched,
* where it is intended to go,
* and what execution mode is requested.

The lifecycle becomes:

```text
Prompt Revision Record
↓

Human Decision Record
↓

Dispatch Record
```

---

# Architectural Review

Leira now has explicit human decisions.

The next boundary is handoff.

A dispatcher must eventually execute work, but before execution exists, Leira needs a deterministic record of intended dispatch.

This keeps execution from becoming hidden authority.

The Dispatch Record says:

> "A human decision authorized this intended handoff."

Nothing more.

# Public API

Create immutable dataclass:

```text
DispatchRecord
```

Fields:

```text
dispatch_id
human_decision_id
subject_id
subject_kind
dispatch_type
target_label
execution_mode
reason_codes
source_label
dispatch_summary
```

All fields are caller supplied.

No generated identifiers.

No timestamps.

No clocks.

No UUIDs.

No randomness.

Unknown dispatch_type values are preserved exactly.

Unknown execution_mode values are preserved exactly.

# Steward's Law

Keep it boring.

The kernel may record that work is intended to be dispatched.

The kernel must not yet perform the dispatch.
