Implement Leira v2.8: CLI Views

Mission

Extend Leira from a machine with internal views into a machine with human-facing commands.

The goal is not a dashboard.

The goal is not a service.

The goal is not a TUI.

The goal is not control.

The goal is to expose existing evidence through deterministic, scriptable commands.

The machine must be able to say:

> Show me the current state.
>
> Show me the audit report.
>
> Show me receipts.
>
> Show me proposals.
>
> Show me sessions.
>
> Show me minds.
>
> No state was changed.

Nothing more.

Hard Constraints

* Python only.
* Single process only.
* Blocking only.
* Library first.
* CLI second.
* No daemon.
* No server.
* No HTTP.
* No websockets.
* No background threads.
* No event creation.
* No dispatch.
* No shell execution.
* No repairs.
* No notifications.
* No subscriptions.
* No progress bars.
* No colors required.
* No interactive prompts.
* No dynamic command discovery.
* No plugin loading.
* No agent loops.

Do not redesign:

* ledger
* projections
* status view
* audit report
* receipts
* proposals

* archive
* workspace
* provenance

Adapt existing structure minimally.

Implementation

Use Python standard library:

```python
argparse
```

Prefer a flat command structure.

Do not implement dynamic subcommands.

Do not scan modules.

Register commands explicitly.

Example structure:

```python
def main():
    parser = argparse.ArgumentParser(prog="leira")
    subparsers = parser.add_subparsers(dest="command")

    status_parser = subparsers.add_parser("status")
    status_parser.set_defaults(func=handle_status)

    audit_parser = subparsers.add_parser("audit")
    audit_parser.set_defaults(func=handle_audit)

    ...
```

Commands

Implement:

```text
leira status
```

Uses only:

```python
get_status_view()
format_status_view()
```

Implement:

```text
leira audit
```

Uses only:

```python
generate_audit_report()
format_report()
```

Implement:

```text
leira receipts <intent_id>
```

Uses existing receipt APIs.

Implement:

```text
leira proposals
```

Uses proposal projections.

Implement:

```text
leira sessions
```

Uses session projections.

Implement:

```text
leira minds
```

Uses mind projections.

Implement:

```text
leira help
```

or equivalent argparse help output.

Rules

Commands are read-only.

Commands do not mutate state.

Commands do not rebuild projections.

Commands do not repair.

Commands do not dispatch workers.

Commands do not invoke minds.

Commands do not execute shell commands.

Commands do not submit inbox intents.

Commands do not acquire claims.

Commands do not approve proposals.

Commands do not walk the ledger when projections already exist.

Import Boundary

The CLI should avoid importing:

```text
dispatcher
proposal_approval
shell adapter
claim store
worker implementations
```

Prefer importing only:

```text
views
audit
receipts
projections
```

The CLI should be structurally incapable of performing work.

Formatting

Output should be:

* deterministic
* stable
* plain text
* human readable
* grep-able
* diff-able

No ANSI colors required.

No progress bars.

No interactive prompts.

No recommendations.

No urgency.

No repair suggestions.

No speculation.

Determinism

Running:

```text
leira status
```

twice against an unchanged database should produce byte-for-byte identical output.

Running:

```text
leira audit
```

twice with fixed timestamps should produce identical output.

Failure Model

Missing intent is data.

Empty system is valid.

Audit failures are valid.

Corrupt projections are valid.

Unknown session is data.

CLI reports.

CLI does not fix.

Kernel Rules

CLI is a view.

CLI is not authority.

CLI exposes existing APIs.

Nothing more.
