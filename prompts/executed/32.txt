# Leira v2.1 — Flow Policy Projection

## Mission

Implement the first deterministic **Flow Policy Projection**.

This is **not** a planner.

This is **not** an AI.

This is **not** a dispatcher.

This is **not** human judgement.

It is a deterministic read model that derives the **currently permitted lifecycle action** from:

* a Lifecycle Projection
* an explicit Flow Policy

It does not invent work.

It does not choose between alternatives.

It merely evaluates deterministic policy rules.

---

# Architectural Review

Leira now possesses:

* immutable evidence
* immutable lifecycle projections

However, every caller must still manually determine:

> "Given this lifecycle state, what action is currently permitted?"

This knowledge should become deterministic.

The Flow Policy Projection answers:

> "According to the supplied policy, what lifecycle action is currently available?"

It never answers:

> "What is the best thing to do?"

That remains human judgement.

---

# Steward's Law

Keep it boring.

The Flow Policy Projection may evaluate explicit rules.

It must never invent new ones.

# Public API

Create immutable dataclasses.

## FlowPolicy

Fields:

```text
policy_name

rules
```

`rules` is a caller-supplied ordered tuple.

No defaults.

No hidden rules.

---

## FlowRule

Fields:

```text
rule_id

required_completed

required_missing

result_action
```

All fields caller supplied.

The projection evaluates rules strictly in caller order.

---

## FlowPolicyProjection

Fields:

```text
subject_id

policy_name

matched_rule_id

recommended_action

evaluation_trace
```

All immutable.

No generated identifiers.

No timestamps.

No clocks.

No randomness.

---

# Evaluation Rules

Input:

* LifecycleProjection
* FlowPolicy

Evaluation:

Walk the rules **in caller order**.

The first rule whose requirements are satisfied becomes the selected rule.

If no rule matches:

```text
recommended_action = "NO_MATCH"
matched_rule_id = None
```

No inference.

No fallback heuristics.

No ranking.

No scoring.

# Design Constraint

The Flow Policy Projection operates **only** on:

* the supplied LifecycleProjection
* the supplied FlowPolicy

It must never inspect evidence directly.

It must never reconstruct lifecycle state.

It must never load policy from disk.

Everything required for evaluation is supplied explicitly by the caller.

---

# Why This Slice Exists

This is the first deterministic interpretation layer in Leira.

Evidence describes reality.

Lifecycle Projection summarizes reality.

Flow Policy Projection determines which lifecycle action is permitted according to explicit policy.

Future Human Decision Records and Dispatcher layers will consume this projection.

The Flow Policy Projection remains purely derived, deterministic, inspectable, and replayable.
