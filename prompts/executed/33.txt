# Leira v2.2 — Missing Evidence Projection

## Mission

Implement the deterministic **Missing Evidence Projection**.

This is **not** a planner.

This is **not** a dispatcher.

This is **not** a recommendation engine.

This is **not** a flow policy.

It is a pure read model that derives the set of lifecycle evidence that has not yet been produced.

Its purpose is to answer one deterministic question:

> "Given the current Lifecycle Projection, which evidence objects are still missing?"

Nothing more.

---

# Architectural Review

Leira now has:

* immutable evidence records
* a Lifecycle Projection
* a Flow Policy Projection

The Lifecycle Projection describes the complete state.

However, callers still have to inspect that projection to determine which evidence is absent.

That should become an explicit deterministic projection.

The Missing Evidence Projection does **not** decide what should happen next.

It merely derives what evidence is currently absent.

Future Flow Policies may evaluate this projection, but this projection itself remains entirely descriptive.

---

# Steward's Law

Keep it boring.

The projection may answer:

> "What evidence is missing?"

It must never answer:

> "What should be created?"

# Design Constraint

The Missing Evidence Projection operates **only** on a supplied LifecycleProjection.

It must never inspect evidence directly.

It must never reconstruct lifecycle state.

It must never scan the repository.

It must never load artifacts from disk.

Everything required for derivation is supplied explicitly by the caller.

---

# Why This Slice Exists

This is the second deterministic interpretation layer in Leira.

Evidence describes reality.

Lifecycle Projection summarizes reality.

Missing Evidence Projection derives what evidence has not yet been produced.

Future Flow Policy Projections may consume this projection to determine which lifecycle actions are currently permitted.

The Missing Evidence Projection remains purely derived, deterministic, inspectable, and replayable.
