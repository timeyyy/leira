# Leira v2.3 — Engineering State Projection

## Mission

Implement the deterministic **Engineering State Projection**.

This is **not** a planner.

This is **not** a dispatcher.

This is **not** a workflow engine.

This is **not** a recommendation engine.

It is a deterministic read model that composes the existing projections into a single engineering snapshot.

The Engineering State Projection answers one question:

> "What is the current engineering state of this subject?"

It performs no judgement.

It performs no execution.

It merely combines existing deterministic projections into a unified view.

---

# Architectural Review

Leira now possesses multiple independent deterministic projections:

- Lifecycle Projection
- Missing Evidence Projection
- Flow Policy Projection

Each projection answers a different question.

However, callers currently need to manually combine these projections to understand the current engineering state.

That composition should itself become deterministic.

This slice introduces the first composite read model.

It owns no canonical state.

It derives everything from supplied projections.

---

# Steward's Law

Keep it boring.

The Engineering State Projection may summarize existing projections.

It must never infer new facts.

It must never recommend actions.

It must never execute work.

# Design Constraint

The Engineering State Projection operates **only** on supplied projections.

It must never inspect evidence directly.

It must never reconstruct lifecycle state.

It must never evaluate flow policy.

It must never load artifacts from disk.

Everything required for composition is supplied explicitly by the caller.

---

# Why This Slice Exists

This is the first composite projection in Leira.

Individual projections answer individual questions.

The Engineering State Projection provides a deterministic engineering snapshot by composing those projections without introducing new behaviour.

This creates a single, inspectable view of engineering state that future Human Decision, Dispatcher, and user interface layers can consume.

It remains purely derived, deterministic, replayable, inspectable, and free of hidden authority.
