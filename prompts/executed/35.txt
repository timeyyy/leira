# Leira v3.0 — Engineering Kernel

## Mission

Implement the first deterministic Engineering Kernel.

This is **not** a planner.

This is **not** a dispatcher.

This is **not** an agent.

This is **not** autonomous.

It is a deterministic coordinator that composes existing projection modules into a single executable pipeline.

It owns no canonical state.

It performs no judgement.

It performs no mutation.

---

## Files

Create:

```text
leira/engineering_kernel/
    __init__.py
    kernel.py
    test_engineering_kernel.py
```

Modify nothing else unless strictly necessary.

---

## Public API

Create immutable dataclass:

```text
EngineeringKernelResult
```

Fields:

```text
subject_id

subject_kind

lifecycle_projection

missing_evidence_projection

flow_policy_projection

engineering_state_projection
```

No generated identifiers.

No timestamps.

No clocks.

No UUIDs.

No randomness.

---

## Functions

Implement:

```text
run_engineering_kernel(...)
```

Inputs:

- caller supplied evidence
- caller supplied FlowPolicy

Pipeline:

1. Build Lifecycle Projection.
2. Build Missing Evidence Projection.
3. Evaluate Flow Policy Projection.
4. Build Engineering State Projection.

Return:

```text
EngineeringKernelResult
```

---

Implement:

```text
engineering_kernel_markdown(...)
```

Returns deterministic markdown.

---

Implement:

```text
write_engineering_kernel(...)
```

Writes:

```text
.leira/engineering_kernel/<subject_id>.kernel.md
```

Derived output only.
