# Leira v4.2 — Execution Adapter Contract

## Mission

Define the deterministic contract between the **Dispatcher Kernel** and all future execution layers.

This is **not** an execution engine.

This is **not** orchestration.

This is **not** scheduling.

This is **not** autonomy.

This is **not** a worker implementation.

The Execution Adapter Contract defines the immutable interface that every execution adapter must satisfy.

It performs no execution.

---

# Context

The following components already exist and are considered complete:

* Dispatch Draft
* Dispatch Commit
* Dispatcher Kernel
* Dispatcher Kernel Receipt Layer

Treat these as stable.

Do not redesign them.

This prompt introduces the next deterministic boundary:

Nothing may execute beyond this boundary yet.

# Public API

Create immutable dataclasses.

## ExecutionCapability

Fields:

```text
adapter_label
adapter_kind
supported_dispatch_types
supported_subject_kinds
supported_execution_modes
supports_parallel_execution
supports_dry_run
supports_interactive_execution
```

---

## ExecutionIntent

Fields:

```text
dispatch_id
adapter_label
dispatch_type
subject_kind
execution_mode
target_label
```

This is a pure projection from the DispatchPlan.

No execution metadata.

No runtime state.

---

## ExecutionAdapterResult

Fields:

```text
dispatch_plan
execution_intent
execution_capability
```

No status.

No success/failure.

No timestamps.

No execution outcome.

# Steward's Law

Keep it boring.

The Dispatcher Kernel determines what would be dispatched.

The Execution Adapter Contract determines what an adapter claims it can accept.

Neither may execute anything.

Execution remains the responsibility of a future execution layer.
