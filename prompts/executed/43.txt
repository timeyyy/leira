# Leira v4.3 — Execution Adapter Compatibility Check

## Mission

Implement a deterministic compatibility check between a `DispatchPlan` and an `ExecutionCapability`.

This is **not** execution.

This is **not** orchestration.

This is **not** scheduling.

This is **not** autonomy.

It only determines whether a declared adapter capability can accept a dispatcher plan.

---

Leira already contains:

* Dispatcher Kernel
* Dispatcher Kernel Receipt Layer
* Execution Adapter Contract

Treat these as complete.

Do not redesign them.

## Compatibility Rules

A dispatch plan is compatible only if:

```text
dispatch_plan.dispatch_type in execution_capability.supported_dispatch_types
dispatch_plan.subject_kind in execution_capability.supported_subject_kinds
dispatch_plan.execution_mode in execution_capability.supported_execution_modes
dispatch_plan.target_label == execution_capability.adapter_label
```

If all rules pass:

```text
compatible = True
reason_codes = ("compatible",)
```

If one or more rules fail, use deterministic reason codes in this exact order:

```text
unsupported_dispatch_type
unsupported_subject_kind
unsupported_execution_mode
target_label_mismatch
```

Example:

```text
compatible = False
reason_codes = ("unsupported_subject_kind", "target_label_mismatch")
```

## Steward's Law

Keep it boring.

The contract says what an adapter claims it can accept.

The compatibility check says whether a dispatch plan matches that claim.

Neither may execute anything.
