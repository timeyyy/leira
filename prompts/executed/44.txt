# Leira v4.4 — Execution Adapter Selection Kernel

## Mission

Implement the deterministic **Execution Adapter Selection Kernel**.

This kernel is responsible for selecting which declared execution adapters are compatible with a `DispatchPlan`.

It is **not** an execution engine.

It is **not** orchestration.

It is **not** scheduling.

It is **not** planning.

It is **not** a worker manager.

It performs no execution.

It merely computes the deterministic set of compatible adapters.

---

# Selection Rules

Selection consists only of partitioning.

If three adapters are compatible:

```text
compatible_adapters = (A, B, C)
```

If two are incompatible:

```text
incompatible_adapters = (D, E)
```

Nothing else.

Do not choose "the best."

Do not reorder.

Do not rank.

Do not score.

Do not infer.

The caller decides what to do next.

# Steward's Law

Keep it boring.

The Dispatcher Kernel determines what would be dispatched.

The Execution Adapter Contract describes what an adapter claims it can accept.

The Compatibility Layer determines whether a dispatch plan matches that claim.

The Selection Kernel partitions adapters into compatible and incompatible sets.

None of these layers may execute anything.
