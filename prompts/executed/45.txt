# Leira v4.5 — Execution Request Layer

## Mission

Implement the deterministic **Execution Request Layer**.

This layer converts a selected adapter into an immutable execution request.

It performs **no execution**.

It is **not** orchestration.

It is **not** scheduling.

It is **not** a worker.

It is **not** an adapter implementation.

Its sole responsibility is to describe exactly what an execution adapter would receive.

---

## Context

The following layers already exist and are considered complete:

* Dispatch Draft
* Dispatch Commit
* Dispatcher Kernel
* Dispatcher Kernel Receipt Layer
* Execution Adapter Contract
* Execution Adapter Compatibility Check
* Execution Adapter Selection Kernel

Treat these as stable.

Do not redesign them.

# Request Rules

The request contains only deterministic information already present in:

* DispatchPlan
* ExecutionCapability

No runtime values.

No environment inspection.

No filesystem inspection.

No repository scanning.

No generated identifiers.

No timestamps.

# Steward's Law

Keep it boring.

The Selection Kernel determines which adapters are compatible.

The Execution Request Layer determines exactly what a compatible adapter would receive.

Neither layer may execute anything.
